2. Prompt 1: Endpoint Documentation
Endpoint: GET /api/products
Purpose: Retrieves a paginated list of products. Supports flexible query filtering by category, price range, and stock availability, along with custom sorting.

Query Parameters:

category (String, Optional): Filter by exact category name.

minPrice (Number, Optional): Minimum price threshold.

maxPrice (Number, Optional): Maximum price threshold.

inStock (Boolean, Optional): If 'true', strictly returns products with a stock > 0.

sort (String, Optional): Field to sort results by. Default: createdAt.

order (String, Optional): Sort direction (asc or desc). Default: desc.

page (Number, Optional): Pagination page number. Default: 1.

limit (Number, Optional): Items per page. Default: 20.

Authentication: None required. Public endpoint.

Responses:

200 OK: Returns a JSON object containing the products array and a pagination metadata object.

500 Internal Server Error: { "error": "Server error", "message": "Failed to fetch products" }

Example Request: GET /api/products?category=electronics&minPrice=50&inStock=true

4. Prompt 3: Developer Usage Guide
How to Use the Product API Endpoint
Welcome! This guide will help you quickly fetch product data using our REST API.

1. Authentication
Good news—this endpoint is completely public. You do not need to attach any API keys or Bearer tokens to your request headers.

2. Formatting Requests
All filters are applied via URL query parameters. You can stack as many as you need. For example, to find cheap, in-stock electronics, append ?category=electronics&maxPrice=50&inStock=true to the base URL.

3. Interpreting Responses
A successful request (200 OK) will always return an object with two root properties:

products: The array of product objects matching your criteria.

pagination: Metadata containing total, page, limit, and pages. Use this metadata to build frontend pagination controls.

4. Example Request (JavaScript/Fetch)

JavaScript
async function getElectronics() {
  try {
    const response = await fetch('https://api.yoursite.com/api/products?category=electronics&limit=5');
    if (!response.ok) throw new Error('Failed to fetch');
    
    const data = await response.json();
    console.log(`Found ${data.pagination.total} total electronics!`);
    console.log(data.products);
  } catch (error) {
    console.error("API Error:", error.message);
  }
}
